# 安全政策

## 支援範圍

目前安全修正以最新的 `main` 分支為準。專案尚未發布穩定版，不承諾舊版分支持續取得修正。

## 回報方式

請勿在公開 Issue 附上含個人資料、機密資訊或受保護內容的 PDF。建立 GitHub repository 後，應啟用 Private vulnerability reporting，並透過 Security 頁面私下回報：

- 問題影響與可重現步驟
- 受影響的瀏覽器與版本
- 最小化、無敏感資料的測試檔案
- 建議修正方式（如有）

## 信任邊界

- PDF 及拆分結果僅在目前瀏覽器工作階段的記憶體中處理。
- 應用程式沒有文件上傳 API，也不使用分析或追蹤服務。
- 網站程式與套件由 GitHub Pages 提供；使用者仍應確認部署來源可信。
- PDF 是不受信任的輸入，檔案格式、大小、頁數及解析錯誤均需在使用前驗證。

## 維護者檢查清單

1. 私下確認與重現問題。
2. 評估是否會造成資料外洩、任意程式執行、資源耗盡或持久化污染。
3. 先準備修正與回歸測試，再公開技術細節。
4. 執行 `npm.cmd run lint`、`npm.cmd run test`、`npm.cmd run build` 與 `npm.cmd run test:e2e`。
